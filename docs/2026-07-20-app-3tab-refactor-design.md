## 一、设计背景

### 1.1 旧版 4Tab 布局问题

旧版 App 底部 Tab 为 4 个：

| Tab | 名称 | 内容 |
|-----|------|------|
| morning | 早点听 | 晨报卡片 + 功能入口网格 + 重磅事件跟踪 |
| insight | 洞察 | 机构调研热门股 + 趋势股评分 |
| forecast | 业绩 | 业绩预测 + 研报 |
| alert | 特别提醒 | 堆叠卡片式异动时间线 |

存在以下问题：

1. **功能交叉**：早点听里混入了异动捕手（个股级功能），洞察里混入了趋势股评分（选股工具），分类逻辑混乱
2. **洞察 Tab 定位不清**：既不是大盘分析，也不是纯选股工具，用户难以形成清晰心智
3. **业绩 Tab 内容单薄**：只有预测和研报两个子模块，单独占一个 Tab 有点浪费
4. **特别提醒交互过重**：堆叠卡片式设计虽然创新，但用户操作成本高，不适合日常快速浏览

### 1.2 设计目标

- 按"信息层级"重新组织 Tab：大盘板块 / 选股工具 / 个股提醒
- 每个 Tab 功能不交叉，用户一眼就知道去哪里找什么
- 短中长线维度只在个股详情页体现，不在 Tab 层级引入
- 保持组件解耦，便于团队并行开发

## 二、设计方案

### 2.1 新版 3Tab 布局

| Tab | 名称 | 定位 | 包含功能 |
|-----|------|------|---------|
| morning | 早点听 | 大盘 + 板块 | 晨报/晚报卡片、长线风口、事件传导、大盘溯源、今日分析概览、重磅事件跟踪 |
| stock | 选股 | 选股工具 | 趋势股评分、机构调研热门股、业绩预测/报告 |
| alert | 提醒 | 个股监控 | 异动捕手、个股情报 |

### 2.2 设计思路

#### 早点听 Tab（大盘 + 板块）

**定位**：用户每天打开 App 第一眼看到的内容，聚焦大盘和板块层面的信息。

**内容布局**（从上到下）：

1. **晨报/晚报卡片**：保留原有设计，显示当日线索数量和摘要标签，点击进入详情页，支持专属播报
2. **功能入口 2x2 网格**：
   - 长线风口 → 板块龙头股预览（跳转 leaders 页面）
   - 事件传导 → 产业链追踪（跳转 event/list 页面）
   - 大盘溯源 → 市场异动溯源分析（跳转 agent-report 页面）
   - 今日分析概览 → Agent 报告更新状态（跳转 agent-report 页面）
3. **重磅事件跟踪**：独立卡片，显示当前最热门的事件标题，点击查看资讯详情

**设计要点**：
- 四宫格每个卡片显示 3 条预览数据，避免信息过载
- "大盘溯源"和"今日分析概览"数据源分离：溯源显示异动分析结果，概览显示各 Agent 报告的更新状态
- 重磅事件作为独立卡片保留，强调其重要性

#### 选股 Tab（选股工具）

**定位**：用户主动选股时使用的工具集合，三大维度横向排列。

**内容布局**（从上到下）：

1. **趋势股评分卡片**：显示前 3 名股票评分排行，带涨跌趋势指示
2. **机构调研热门股卡片**：显示前 3 名机构调研最密集的股票
3. **业绩预测卡片**：显示前 3 名股票的业绩预测数据

**设计要点**：
- 三张卡片使用不同颜色装饰条区分（蓝/橙/绿）
- 每张卡片头部带图标 + 标题 + 描述 + 更多箭头
- 卡片内嵌白色预览列表，显示排名、名称、标签/分数
- 底部"查看详情"按钮跳转到对应详情页
- 卡片宽度优化：padding 从 28rpx 减至 20rpx，字号整体缩小，preview-name 加溢出控制，避免横向滑动

#### 提醒 Tab（个股监控）

**定位**：用户关注的个股相关提醒和情报，从堆叠卡片改为列表式快速浏览。

**内容布局**（从上到下）：

1. **异动捕手模块**：
   - 显示 4 种异动类型：涨停封板（封）、急速下跌（速）、异常放量（量）、快速拉升（涨）
   - 每条显示股票名、异动详情、时间戳
   - 点击"实时监控"跳转 event-catcher 页面
2. **个股情报模块**：
   - 支持"全部/利好/利空"标签切换筛选
   - 情报来源分类：公告（公）、研报（研）、新闻（新）
   - 每条显示标题、来源标签、情感倾向、时间

**设计要点**：
- 移除原堆叠卡片的长按拖动交互，改为直接列表展示，降低操作成本
- 异动类型用彩色 badge 区分（红/橙/蓝/绿），一眼可辨
- 情报模块支持情感筛选，帮助用户快速过滤利好/利空

### 2.4 路由检测逻辑

`AppBottomBar.vue` 的 `activeTabId` 计算属性根据当前页面路由自动激活对应 Tab：

| 路由匹配 | 激活 Tab |
|---------|---------|
| `home/pages/index`（无参数） | morning |
| `home/pages/index?tab=stock` | stock |
| `analytics/pages/index` | stock |
| `market/pages/hot-burst` | stock |
| `analytics/pages/trend-score` | stock |
| `analytics/pages/forecast` | stock |
| `analytics/pages/reports` | stock |
| `favorites/pages/index` | alert |

### 2.5 主 Tab 容器交互

`MainTabs.vue` 统一管理 3 个 Tab 的内容切换：

- 使用 `v-show` 保持组件状态，切换不销毁
- 选股 Tab 保留预测/报告子切换（`stockSubTab` 状态）
- 移除原特别提醒的底部操作栏（alertIdx、alertTotal、syncAlertState 等逻辑）
- `AlertContent` 仍暴露 `currentStockIdx` 和 `totalCount` 以保持接口兼容

## 五、设计决策记录

### 5.1 为什么不加"我的"Tab？

用户明确反馈不需要"我的"页面。个人中心、设置等功能通过其他入口（如设置图标）进入，保持底部 Tab 聚焦在核心业务功能。

### 5.2 为什么把"异动捕手"从早点听移到提醒？

异动捕手是"个股级"监控功能，和自选股异动、个股情报属于同一层级。放在早点听（大盘板块层级）会导致功能交叉，违反"Tab 间功能不交叉"原则。

### 5.3 为什么把"溯源Agent"改名为"大盘溯源"？

- "Agent"是技术术语，普通用户不理解
- "大盘溯源"更直白地表达了功能定位：追溯大盘异动的源头
- 与"事件传导"（板块级）、"重磅事件"（宏观级）形成清晰的大盘分析矩阵

### 5.4 为什么恢复"重磅事件跟踪"独立卡片？

四宫格空间有限，重磅事件需要更醒目的展示位置。独立卡片可以显示完整事件标题和"点击查看资讯详情"的引导，比挤在四宫格里更有效。

### 5.5 为什么选股 Tab 保留预测/报告切换？

业绩预测和研报是两种不同的分析视角，放在同一个 Tab 下通过子标签切换，避免增加 Tab 数量，同时保持切换便捷性。
